I believe it is an imporovement from a readability and robustness standpoint. By replacing raw pointers with shared_ptr, I can eliminate manual new/deletes, null-inits, and destructors, since memory is managed automatically by shared_ptr, including initialization and cleanup. I was also able to remove the tree constructor since no more null inits were needed. I had originally used unique_ptr, but ran into ownership problems with search(). unique_ptr would have also made the copy() method must less straightforward to implement. shared_ptr made traversing and copying the tree recursively much more straightforward than using raw pointers. In addition, I also added a node constructor method using a member initializer list in order to avoid partial initalization problems. Overall, using RAII abstractions like shared_ptr makes the implemenation of the binary tree safer and more readable.
